## 🚀 Deputy Deployment & Installation Guide

If you are setting this system up on a new GitHub environment or expanding local code files, follow these deployment guidelines:

### 1. Repository Setup
1. Initialize a **Public** repository named `lasd-database` (or your preferred directory name).
2. Upload all `.html` asset files directly to the root directory `( / )`.
3. Ensure your file naming structures completely match the lowercase nomenclature listed in the matrix above (e.g., `policy.html`, `bwc.html`).

### 2. Activating GitHub Pages
1. Navigate to the ⚙️ **Settings** tab located at the top toolbar of your repository.
2. Select **Pages** from the left navigation sidebar menu under *Code and automation*.
3. Locate **Build and deployment** -> **Source** and verify it is configured to `"Deploy from a branch"`.
4. Under the **Branch** dropdown, change the pointer from `None` to `main` (or `master`), preserve the folder configuration as `/ (root)`, and click **Save**.
5. Your custom deployment URL will generate at the top of that menu block within 1–2 minutes.
